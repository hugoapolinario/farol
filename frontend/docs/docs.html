Wrap your agent

Replace frl_your_key_here with your API key and my-agent with your agent's name. Then wrap your existing function with @trace and add token tracking inside your LLM call. 
from farol import trace
from anthropic import Anthropic

client = Anthropic()

@trace(agent_name="my-agent", farol_key="frl_your_key_here")
def my_agent(task, run=None):
    run["topic"] = task

    with run.span("llm_call", type="llm") as span:
        response = client.messages.create(
            model="claude-haiku-4-5",
            max_tokens=1024,
            messages=[{"role": "user", "content": task}]
        )
        span.input_tokens = response.usage.input_tokens
        span.output_tokens = response.usage.output_tokens

    return response.content[0].text

                "model": "claude-haiku-4-5",
                "max_tokens": 1024,

        span.output_tokens = response["usage"]["output_tokens"]

    return response["content"][0]["text"]

Run your agent normally

agent.py
# Just call your function as usual — Farol tracks everything automatically
result = my_agent("your task here")

Every call to your agent will now appear in your Farol dashboard automatically.

Open your Farol dashboard — you'll see the run appear within seconds, with cost, duration, token usage and any errors automatically tracked.

SDK reference

The trace decorator wraps your function and passes a run dict for steps, tokens, and timing.

ParameterDescription
agent_name Required. Display name for this agent in the dashboard.
farol_key Required. API key from the dashboard; ties runs to your account.
model Optional. Model label for display. Default "unknown".
cost_per_1k_tokens Optional. USD per 1k tokens (input + output) for cost. Default 0.00025.

Dashboard

Each row is one run. Columns:

  • Agent — name you set with agent_name.
  • Topic — optional topic string if you set run["topic"].
  • Status — success or error for that run.
  • Anomaly — cost spike vs your recent baseline, or “baseline” while Farol learns.
  • Steps — number of recorded steps in the trace.
  • Duration — wall-clock time for the run.
  • Tokens — input and output token counts.
  • Cost — estimated USD from tokens and cost_per_1k_tokens.

FAQ

Does Farol store my prompts?

Farol is built for run metadata: timing, tokens, cost, steps, and errors. Do not put secrets in fields you sync. Treat prompt content like any production telemetry—only send what you are comfortable storing in your database.

Which frameworks does Farol support?

Any Python code path: the SDK is a decorator. LangChain, CrewAI, custom loops—wrap the entrypoint and update run with token counts (and optional steps) however your stack exposes them.

How does cost anomaly detection work?

After enough successful runs for the same agent, Farol compares the current cost to a rolling baseline (median and spread). Large jumps—relative to that baseline—flag as anomalies so you catch bill spikes early.

Can I use Farol without a Farol key?

The decorator runs without a key, but runs are not tied to your dashboard and are not synced under your account. Pass farol_key from the app to attribute runs to you.
